**Prompt library — direction for after synthesis is built:**
You asked about this — here's a recommendation, not yet built. Two different things could be called a "prompt library" here, worth keeping separate:
- **Specialist system prompts** (already exist, in code) — these are the brand-bounded reasoning rules in each `src/specialists/*.js` file. Not user-facing; you don't write these per-use.
- **User-facing task prompts** (doesn't exist yet, this is the real "library") — reusable templates *you* would paste in when using the finished tool for different situations: e.g. "full project review" (run all detected brands, produce all 6 sections), "measure-day-only checklist for [brand]", "submittal risk scan for [brand] against these shop drawings", "installation briefing handoff for the field crew." These map to *how you actually use the tool day to day*, not to the underlying agent logic.

Recommendation once the synthesis agent exists: build this as a small `Docs/Prompt_Library.md` (or a `prompts/` folder if you want them as separate copy-pasteable files) with maybe 5-8 templates covering your actual recurring PM workflows — preconstruction review, measure day, submittal review, installation handoff, multi-brand project kickoff. Keep it thin at first; add templates only when you notice yourself re-typing the same framing more than twice. Don't try to anticipate every use case up front — the six report sections already impose most of the structure you need.
